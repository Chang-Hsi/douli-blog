## 前言

寫這篇文章時其實有點猶豫，我才做這行兩年，真有資格寫下這篇文嗎？但想想反正這篇文更多是寫給自己看，就當是對自己這一路走來的技術成長做個回顧吧。

## 轉職起始

大學時唸的是中國文學系跟法律系雙主修，就如自我介紹所說，我是個重度選擇症患者，當不知道要選哪個的時候，當前的我選擇兩個都要。

畢業出來以後，考了兩年國考，可惜都沒成功。遂決定在家附近找找補習班，去應徵補教老師，我從大學時就給人當家教，對於教別人學習還是有點自信的，也確實在補教界當了幾年老師，然而好景不長，因為疫情緣故，這行越來越難做。

於是在找能夠遠端的工作，在朋友的建議下兩人一起報名桃園職訓局開的後端 JAVA 課程，當時我想說去試試看也行，就當多掌握一門技能也不錯，那時候根本沒想到我會在這行一直做下去。

轉職的契機在就業媒合會後不久，有家新創公司來找我，詢問我是否有意向去他們公司擔任前端工程師，對我來說這是天堂開局，省略去 104 投履歷的流程，當時就確定錄取，於是 2023 / 09 / 01 我正式轉職。

## 第一個專案

剛進公司時，其實對於自己的工作處於一種茫然的狀態，不知道要做什麼，要怎麼做，做好以後怎麼提交等等等，當然的作法就是問，問同事、問老闆、問 GPT、問 Stack overflow 等等。

工作了一個月，投入了莫大的精力與時間，終於勉強理解了一個上線的後台管理系統是怎麼回事，從語言到框架，從共用組件到頁面功能，跳轉頁面依靠的機制，狀態管理工具 Pinia，如何對 UI 框架的進行高度的客制化等。

現在回想起來，一方面為自己因技術不成熟抱有的一些觀念感到好笑，一方面又覺得自己能熬過來還算是有點厲害。

## 第二個專案

第二個專案也是接手前人開發的專案，但這次的性質比較不同。屬於前面的公司搞砸了，我們充當救火隊接手，專案的開發時程非常短，測試週期也短。而且使用的框架是 Nuxt + Element Plus，是從自己我意識到不同框架的能帶來的意義，還有專案的寫法是沒有固定解的。

比如説，因為 Nuxt 的 `pages/` 目錄下的檔案結構會自動轉換成 Vue Router 路由，因此專案的目錄，文件的命名風格等，跟 Vue 會有所區別。一開始確實不太適應，但習慣以後就覺得這樣也挺好的。

再來就是，因為案子非常緊急，第一次跟其他前端共同開發，那時為了避免專案之間產生 merge conflict，對 Gitlab 建分支，發合併請求，解 conflict 等也是增進不少實戰經驗。

附帶一提也是這時候接觸到 Tailwind 的，為了加快切版效率，資深前端過來給我培訓 Tailwind 的用法，讓我有空就去練習，從一開始我對 Tailwind 的抗拒，到現在的真香，回想起來也是覺得很有意思。

## 第三個專案

大約是 2024 年 10 月底，我被任命負責主導與開發新案子，這是個洗衣平台的案子，大體來說是要前台做 Web 應用程式，後台管理系統，POS 負責儲值、結帳，建訂單，轉廠跟配送等。

我底下有三個員工，一個資深後端，一個資深前端，一個新人前端。這對我來說又是一項新挑戰，之前我只需要顧好自己的 code 就好，現在我不僅要顧好自己的 code，還要管別人，一時間壓力山大。

為了方便管控進度，我建立 Google Sheet，制定開發任務跟時程，追蹤每天進度，如果有人進度卡住要去處理，資深組我其實我沒有太費心，他們自己能把自己的部分管控好，頂多每天需要追蹤一下進度，當時給我壓力最大的是新人（笑）。

我想，如果有人問我工作中遇到最大的挑戰是什麼，我應該會說**帶領一個比我更迷茫的人**吧。

我在新人身上看到了剛入行時那個手忙腳亂的自己，但我卻沒有多餘的精力去包容他。我得在保證專案不開天窗的前提下，一邊保證自己跟資深組這邊的進度沒問題，一邊指導新人該如何完成他負責的部分。

我認為帶人最難的不是技術，而是那種「責任感」帶來的窒息感。我不能再說「我不會」，我不能跟上級説，因為新人不知道怎麼做，所以案子沒在時間內完成任務進度，因為我就是負責管理這個，我的職責就是避免任務開天窗。

## 總結

這一篇算是幫自己做了一次職涯的回顧，我覺得比起回憶自己擁有哪些技術棧什麼的，分享一下這兩年來的回顧會更有用一點，可以明確地看到自己有在進度，也能明顯感覺到自己還有很大的進步空間。

我希望自己能成為更好的人，希望有一天能自稱自己已經是一個資深者，希望能掌握更多元的前端技能，希望自己的技能樹能往超過前端，往全端發展。

> 希望有一天當我回顧這篇文章時，也能為我當下的稚嫩會心一笑。
